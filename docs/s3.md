# S3

> S3 es un servicio de almacenamiento de objetos, el cual consiste en almacenar datos como objetos dentro de lo que se denominan buckets el cual es un contenedor de objetos.

## Desarrollo

Para el desarrollo se utiliza la imagen de docker Garage, el cual satisface las necesidades de tener un servidor compatible con la API de S3, de tal forma que se evita la neceidad de configurar un bucket en AWS.

### Configuración de Garage
Garage requiere de la configuración de variables de entorno para su funcionamiento, las cuales se encuentra en `docker/.env` y configurar según sea necesario.

Garage tiene dos carpetas, la primera es meta que es donde se almacenan los metadatos de los objetos, y la segunda es data que son los datos de los objetos estas carpetas se encuentran en 'data/s3-garage/meta' y 'data/s3-garage/data' respectivamente.

Para interactuar con el contenedor de garage una vez levantado, se puede utilizar el siguiente comando:

```bash
docker exec -it ania_s3_garage /garage <comando>
```

Revisar status: 

```bash 
docker exec -it ania_s3_garage /garage status
```

Este comando nos arrojará un NODE ID, el cual será necesario para el siguiente paso por lo cual copialo.
Ejemplo de salida (El apartado ID es el que nos interesa):

```bash
ID                  Hostname   Address              Tags  Zone  Capacity
a1b2c3d4e5f6789a    ania_s3    172.20.0.104:3901    -     -     NOT CONFIGURED
```

### Creación del cluster layout de garage

Un cluster layout es el apartado en el cual le informamos a garage sobre el espacio disponible para cada nodo del cluster.

> Cluster: Es un conjunto de nodos que trabajan juntos para proporcionar almacenamiento y procesamiento de datos.

> Nodo: En el contexto de S3/Garage, un nodo es una instancia fisica o virtual que representa el donde se encuentra almacenado un dato, es decir, un nodo es un espacio de almacenamiento dentro del cluster.

> Zona: Es una agrupación lógica de nodos dentro de un cluster, que se utiliza para mejorar la disponibilidad y la tolerancia a fallos.

Para crear el cluster layout de garage, se debe ejecutar el siguiente comando para crear un layout con un nodo de 1GB y una zona con nombre dc1:

```bash
docker exec -it ania_s3_garage /garage layout assign -z dc1 -c 1GB <NODE_ID>
``` 
Lo anterior te deberá arrojar algo como esto:

```bash

Use `garage layout show` to view staged role changes,
and `garage layout apply` to enact staged changes.
```

Como menciona el mensaje, el cambio aún no se ha aplicado. Para visualizar el estatus del layout, se puede ejecutar el siguiente comando:

```bash
docker exec -it ania_s3_garage /garage layout show
```

### Creación de credenciales

AWS_ACCESS_KEY_ID y AWS_SECRET_ACCESS_KEY son las variables de entorno que se utilizan para configurar las credenciales de acceso a S3, estas variables se encuentran en `docker/.env` y se pueden configurar según sea necesario.

Creación de las credenciales con nombre `garage_key`:

```bash
docker exec -it ania_s3_garage /garage key create garage_key 
```

Esto nos dará algo como esto, que colocaremos en el archivo `docker/.env` para configurar las credenciales de acceso a S3 de las variables de entorno AWS_ACCESS_KEY_ID y AWS_SECRET_ACCESS_KEY respectivamente:

```bash
==== ACCESS KEY INFORMATION ====
Key ID:              <ESTE ES TU AWS_ACCESS_KEY_ID> 
Key name:            garage_key
Secret key:          <ESTE ES TU AWS_SECRET_ACCESS_KEY> 
Created:             2026-03-21 20:56:42.842 +00:00
Validity:            valid
Expiration:          never

Can create buckets:  false

==== BUCKETS FOR THIS KEY ====
Permissions  ID  Global aliases  Local aliases
```

Ahora elevamos los permisos para que pueda crear buckets:

```bash
docker exec -it ania_s3_garage /garage key allow <AWS_ACCESS_KEY_ID> --create-bucket
``` 

Esto nos dará un mensaje similar al anterior, con la diferencia de que ahora aparece en verdadero el permiso de crear buckets:

```bash
Can create buckets:  true
```

### Creación de bucket
Ahora que tenemos las llaves de acceso, podemos crear un bucket con el siguiente comando:

```bash
docker exec -it ania_s3_garage /garage bucket create ania-s3-bucket
```

Esto nos dará algo como esto:

```bash
==== BUCKET INFORMATION ====
Bucket:          <BUCKET_ID> 
Created:         2026-03-21 21:04:08.110 +00:00

Size:            0 B (0 B)
Objects:         0

Website access:  false

Global alias:    ania-s3-bucket

==== KEYS FOR THIS BUCKET ====
Permissions  Access key    Local aliases
```

Damos permiso de acceso al bucket a la llave que creamos anteriormente:  

```bash
docker exec -it ania_s3_garage /garage bucket allow ania-s3-bucket --key <AWS_ACCESS_KEY_ID> --read --write
```
Esto nos generará un mensaje similar al anterior pero mostrando que ahora tiene permisos de lectura y escritura en la llave.

#### Configuración del docker/.env
Una vez terminado los pasos anteriores ya podemos colocar en el .env las siguientes variables de entorno:

```
AWS_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY
```

### Reinicio del server

Para que los cambios en el .env tengan efecto, es necesario reiniciar el server con el siguiente comando:

```bash
docker compose -f docker/docker-compose.yml -f docker/docker-compose.dev.yml restart server 
```
